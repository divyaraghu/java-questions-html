  15. Difference between POST and PUT
    Always use PUT method to UPDATE an existing resource. PUT methods are idempotent. Even when we call the PUT method multiple times, result will be same.
    POST methods are generally used for CREATING resources. Its not idempotent, so if you call it multiple times, its going to create multiple resources.
  16. Difference between git pull and git fetch
    git pull is git fetch + git merge. git fetch will not merge the incoming changes with local repo, whereas git pull will fetch + merge.
  17. How do you debug, what tools you used for debugging
    Take thread dumps using the command jstack pid
    A thread dump will have the thread name, thread id, priority, thread state and call stack information. Analyse the thread dump, check for any threads with more wait time, check for any dead locks, check for really more number of threads created at the same time that will deplete the CPU (take the tiff file thumbnail example)
    When an application is slow, look for blocked threads. One thread will be holding a shared object, and other threads will be blocked, this will cause the application to slow down.
    Look for any deadlocks - see if 2 thread are waiting for each other's locks.
    Check if any thread is in WAITING state for a long time

  20. What is a memory leak? How to overcome it?
    Java objects holding up a lot of memory and those objects are not being garbage collected leads to memory leak
    1.Using lots of static objects can lead to memory leak. (Especially large static collections). Static objects live in memory as long as the application runs, so it'll not be garbage collected and hence will lead to memory leak
    2. Opening resources, connections and sessions and not closing them will also lead to memory leak. Garbage collector will think those objects are still being used since its open and hence will not garbage collect it.
    To overcome memory leak, minimize use of static objects, always use finally block to close any resources or sessions that is opened. Use lazy loading design pattern where objects are initialized only when called for the first time.
    There are tools like jProfiler and Java Visual VM to identify memory leaks.
  21. How to find tune sql queries?

    1. Use select field names instead of select *
    2. Select * from table1,table2 where table1.id = table2.id. this is called Cartesian join or cross join. Avoid this kind of query, always use inner join with on.
    3. Avoid Select distinct, that's costly, Instead use select with more fields
    4. Use where whenever you use having. Having should only be used on aggrevating results along with where
    5. Use indexing/filters when necessary
    6. Aggregate the data as much as possible
    7. Avoid using OR on inner join condition, it'll read lots of rows even for 2 simple tables, instead use union
  22. How do you make threads be called in a sequence?
    Using join method. Start a thread and call join method on the same thread. It'll make it to wait until the thread stops executing. This way all the threads will be called in order
    t1.start(); t1.join();
    t2.start(); t2.join();
    t3.start();t3.join();
